Detailed Description

Handles communication with the tracktion web-store to register and unlock an application.

To use: create a subclass of this, and implement the pure virtual methods. Then you can instantiate an instance (probably best to just keep a single instance around for the duration of your app) and use its methods to do the various registration tasks.

virtual StringArray TracktionMarketplaceStatus::findAllowedMachineIDs ( )
virtual

Returns a list of strings, any of which should be unique to this physical computer.

When testing whether the user is allowed to use the product on this machine, this list of tokens is compared to the ones that were stored on the Tracktion marketplace webserver.

The default implementation of this method will calculate some machine IDs based on things like network MAC addresses, hard-disk IDs, etc, but if you want, you can overload it to generate your own list of IDs.

The IDs that are returned should be short alphanumeric strings without any punctuation characters. Since users may need to type them, case is ignored when comparing them.

Note that the first item in the list is considered to be the "main" ID, and this will be the one that is displayed to the user and registered with the marketplace webserver. Subsequent IDs are just used as fallback to avoid false negatives when checking for registration on machines which have had hardware added/removed since the product was first registered.
